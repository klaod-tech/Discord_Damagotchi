## 한 줄 소개

먹구름(mukgoorm) — 유저가 음식을 입력하면 나만의 캐릭터에게 밥을 주는 행위로 연결되며, 칼로리·날씨·체중·이메일 정보를 **수치 없이 캐릭터 이미지와 대사로 간접 전달**하는 라이프스타일 관리 웹 애플리케이션.

---

## 핵심 원칙 (절대 변경 불가)

1. **hp/hunger/mood 수치는 사용자에게 절대 직접 노출 금지** — 이미지+대사로만 간접 표현
2. **날씨는 별도 알림 없음** — 기상 시간에 이미지 자동 교체로만 전달
3. **칼로리/영양소 수치는 오늘 요약 클릭 시에만 확인 가능**
4. **각 기능은 자신이 소유한 DB 테이블에만 INSERT/UPDATE**

---

## 전환 배경 — Discord 봇 → 웹앱

### 해결된 문제

| 문제 (Discord 봇) | 해결 (웹앱) |
|-------------------|------------|
| 관리자가 모든 유저 쓰레드 열람 가능 (Discord 구조상 막을 수 없음) | Supabase Auth → 유저별 완전 격리 |
| Discord 임베드 디자인 한계 (이모티콘 외 커스텀 불가) | CSS 자유 활용, 주간 리포트 고품질 |
| 버튼/Modal 기반 UX 제약 | 웹 UI 자유로운 인터랙션 |
| 앱 설치 없이 사용 불가 | 브라우저 접속, 향후 Electron 패키징 가능 |

### Discord의 새 역할

Discord를 완전히 제거하지 않고 **알림·리포트 발송 채널**로 활용.

- 주간 리포트 Discord/이메일 발송
- 일정 Push 알림
- 원하는 경우 이메일 대체 가능

## 기술 스택 (신규 — React 웹앱)

| 항목 | 내용 |
|------|------|
| 언어 | JavaScript / TypeScript |
| 프레임워크 | React (Vite) |
| 데이터 | Supabase (PostgreSQL + Auth) |
| AI | OpenAI GPT-4o-mini |
| 자동화/ML | n8n (워크플로우 · ML 파이프라인) |
| 알림 | Discord Webhook · 이메일 (선택) |
| 이미지 | NovelAI (캐릭터 이미지, 기존 유지) |
| 날씨 | 기상청 공공데이터 API |
| 영양 | 식품의약품안전처 식품영양성분 DB API |
| 칼로리 공식 | Mifflin-St Jeor |

> **왜 JavaScript로 전환?**  
> Claude Code가 JavaScript/TypeScript를 가장 잘 처리함 (AI 학습 데이터 압도적 우위).  
> 코드 품질·속도 모두 JS가 유리. Python 코드를 JS로 변환하면서 진행.

## 전체 아키텍처

```
브라우저 (React 웹앱)
  │
  ├── Supabase Auth      — 유저 인증 · 세션 관리
  ├── Supabase DB        — 데이터 CRUD (기존 스키마 재활용)
  ├── GPT-4o-mini API    — AI 대화 · 칼로리 분석 · 대사 생성
  │
  ├── n8n (별도 서버)
  │     ├── ML 의도 분류 파이프라인
  │     ├── 음식 추천 워크플로우
  │     └── 자동화 트리거
  │
  └── 알림 레이어 (선택)
        ├── Discord Webhook — 주간 리포트 · 일정 Push
        └── 이메일 (SMTP)  — 리포트 발송
```

## 기능 구조

```
웹앱
├── 캐릭터 상태 뷰       — 이미지 + GPT 대사 (메인 화면)
├── 식사 기록            — 텍스트/사진 → 칼로리 분석
├── 체중 관리            — 기록 · 추이 그래프 · 목표 달성
├── 날씨                 — 기상청 API → 캐릭터 이미지 반영
├── 일정 관리            — 등록 · 알림 · 반복
├── 일기                 — 작성 · 감정 분석
├── 이메일 모니터링      — 네이버 IMAP 폴링 · 알림
├── 주간 리포트          — CSS 뷰 · Discord/이메일 발송
└── 음식 추천            — n8n → 주변 식당 + 잔여 칼로리
```

## 작업 순서

```
[1단계] React 기본 세팅
  ─ Vite + React + TypeScript
  ─ Supabase client 연결
  ─ 라우팅 구조 (react-router)
  ─ 기본 레이아웃 (사이드바 + 메인 영역)

[2단계] 인증 · 유저 프로필
  ─ Supabase Auth (이메일 로그인)
  ─ 온보딩 플로우 (이름·도시·목표체중 등록)
  ─ 기존 DB 스키마 재활용

[3단계] 핵심 기능 (JS 변환)
  ─ 캐릭터 상태 컴포넌트 (이미지 + GPT 대사)
  ─ 식사 기록 (텍스트 + 사진)
  ─ 체중 기록 + 추이 그래프
  ─ 날씨 표시

[4단계] 확장 기능
  ─ 일정 관리
  ─ 일기 + 감정 분석
  ─ 이메일 모니터링

[5단계] n8n 연동
  ─ ML 의도 분류 파이프라인 연결
  ─ 음식 추천 워크플로우 연결

[6단계] 알림 · 리포트
  ─ 주간 리포트 CSS 뷰
  ─ Discord Webhook 발송
  ─ 이메일 발송 (선택)
```

## DB 스키마 (기존 Supabase 재활용)

기존 Python 봇이 쌓은 Supabase DB 스키마를 그대로 재활용.  
웹앱에서 동일한 테이블에 접근하므로 마이그레이션 불필요.

| 주요 테이블 | 설명 |
|------------|------|
| `users` | 유저 프로필, 설정, 채널 ID |
| `meal_log` | 식사 기록 (칼로리, 영양소) |
| `weight_log` | 체중 기록 |
| `schedules` | 일정 |
| `diary_log` | 일기 + 감정 |
| `intent_log` | ML 학습용 의도 분류 데이터 |

## 유저 규모 및 설계 기준

| 항목 | 내용 |
|------|------|
| 현재 운영 | 2인 프라이빗 개발 환경 |
| 목표 규모 | 최대 20인 소규모 |

| API 비용 | 20인 × GPT 호출 빈도 — 월 예산 계획 필요 |
| n8n | 별도 서버 또는 클라우드 n8n 사용 |
